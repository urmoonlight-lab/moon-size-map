# Moon Size Map

身形友好的服装尺码与版型路线图。

Moon Size Map 是一个本地优先的服装尺码辅助工具。它帮助用户根据身体数据、商品尺码表和版型信息，找到第一批可试尺码、红线尺寸、可能不合适的位置和试穿优先级。

## 它是什么

- 一个纯前端 PWA，可以直接在浏览器打开。
- 一个用于保存身形记录、商品尺码表、尺码路线和试穿反馈的小工具。
- 一个强调隐私、冷静判断和版型适配的本地记录本。

## 它不是什么

- 不是身体评价工具。
- 不是“你就是 M/L/XL”的身份标签计算器。
- 不判断身形是否标准，也不输出身体比较或分数。

尺码不是身份，版型才是地图。衣服应该适合身体，而不是让身体为粗糙尺码表负责。

## 本地优先数据说明

- 没有后端。
- 没有登录。
- 没有云上传。
- 没有分析追踪。
- 数据保存在当前浏览器的 `localStorage`。

清理浏览器数据、更换设备、卸载浏览器或使用隐私模式，都可能删除记录。建议定期在“数据”页导出 JSON 备份。

## 如何本地运行

直接打开 `index.html` 即可使用。

如果想测试 PWA 和 service worker，建议启动一个本地静态服务器：

```bash
python3 -m http.server 8000
```

然后访问：

```text
http://localhost:8000
```

## GitHub Pages 部署

这个项目是静态站点，可以直接部署到 GitHub Pages：

1. 将文件推送到 GitHub 仓库。
2. 在仓库 Settings 中打开 Pages。
3. 选择 main 分支和根目录。
4. 等待 Pages 生成访问地址。

## 隐私原则

Moon Size Map 不上传用户数据，不做登录系统，也不做云端同步。所有记录默认留在当前浏览器中，用户可以通过 JSON 导出和导入进行备份与迁移。

## 语言原则

界面和结果文案避免身体羞辱、性别羞辱和“标准身材”之类表达。推荐使用：

- 身形友好
- 版型适配
- 试穿路线
- 红线尺寸
- 可能翻车的位置

避免使用：

- 否定身体的标签
- 暗示某种身形才合理的表达
- 性别羞辱或身体比较
- 把尺码结果写成身份结论
